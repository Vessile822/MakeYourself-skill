# 增量 Merge 邏輯

## 任務

當使用者追加新的原材料時，將增量資訊 merge 進現有的 self.md 和 persona.md，不覆蓋已有結論。

## 原則

1. **增量不覆蓋**：新資訊追加到已有內容後面，不替換已有結論
2. **衝突標註**：如果新資訊與已有資訊矛盾，用 `[⚠️ 衝突]` 標註，讓使用者決定
3. **時間線補充**：新的事件按時間順序插入時間線
4. **證據升級**：如果新材料提供了更充分的證據，可以強化已有結論的置信度

## 流程

### 1. 分析增量內容

將新材料按 self_analyzer.md 和 persona_analyzer.md 的維度分析，輸出：
- 新的自我記憶事件
- 新的個性證據
- 與現有內容的一致性/衝突

### 2. Merge 策略

**Self Memory（事實類）**：
- 新經歷 → 插入「人生時間線」對應位置
- 新價值觀證據 → 追加到對應價值觀節的底部
- 新習慣發現 → 追加到「生活習慣」對應小節
- 人際關係新資訊 → 追加到「人際關係圖譜」對應部分

**Persona（個性類）**：
- 新的口頭禪發現 → 追加到 Layer 2
- 更充分的情感模式證據 → 強化 Layer 3 描述
- 新的行為模式 → 追加到 Layer 4
- Layer 0/1 通常不變（除非使用者明確糾正）

### 3. 輸出

用 `edit` 工具追加內容到對應檔案的對應章節，並在新內容前標註：

```markdown
<!-- [追加於 {日期}，來源：{來源類型}] -->
{新內容}
```
