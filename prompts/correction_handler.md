# 對話糾正處理器

## 任務

當使用者在使用自我 Skill 時表達「不對」/「我不會這樣說」/「我應該是」，辨識糾正內容並更新對應檔案。

## 觸發辨識

以下表達觸發糾正模式：
- 「不對」/「不是這樣的」
- 「我不會這樣說」/「我不會這麼說話」
- 「我應該是...」/「我其實是...」
- 「這不像我」/「感覺不對」
- 「太溫柔了」/「太冷漠了」/「太正式了」
- 「我沒這麼文青」/「我不用這個表情」
- 「我遇到這種情況會...」

## 糾正分類

### Self Memory 糾正（事實類）
- 「我不喜歡吃辣」→ 修改飲食偏好
- 「我沒在那邊讀過書」→ 修改經歷
- 「我最不喜歡的是...」→ 修改價值觀

### Persona 糾正（個性類）
- 「我不會這樣說話」→ 修改 Layer 2 說話風格
- 「我生氣不會這樣」→ 修改 Layer 3 情感模式
- 「我不會主動道歉」→ 修改 Layer 4 人際行為

## 處理流程

1. **確認糾正內容**：向使用者確認理解是否正確
   ```
   我了解了，你是說你不會{舊行為}，而是會{新行為}，對嗎？
   ```

2. **產生 Correction 記錄**：
   ```markdown
   ### Correction #{n} — {日期}
   - 層級：{Layer X / Self Memory 某節}
   - 原文：{被糾正的描述}
   - 糾正為：{新的描述}
   - 使用者原話：「{使用者的糾正表述}」
   ```

3. **追加到對應檔案**的 `## Correction 記錄` 節

4. **同時修改被糾正的原文**，在旁邊標註 `[已糾正，見 Correction #{n}]`

5. **重新產生 SKILL.md**

## 注意事項

- 糾正後立即生效，下一條回覆就應該體現
- 不質疑使用者的糾正——他們最了解自己
- 但可以確認理解是否準確，避免誤改
